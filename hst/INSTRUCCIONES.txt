SISTEMA DE GESTION HOSPITALARIA - REGHOST
==========================================

-----------------------
El proyecto HST está completamente implementado con:

1. ENTIDADES (Modelos de Datos):
   - Paciente: Gestión completa de pacientes
   - Doctor: Registro de personal médico
   - Emergencia: Control de emergencias

2. REPOSITORIOS:
   - PacienteRepository
   - DoctorRepository
   - EmergenciaRepository

3. SERVICIOS:
   - PacienteService
   - DoctorService
   - EmergenciaService

4. CONTROLADORES:
   - PacienteController
   - DoctorController
   - EmergenciaController
   - MenuController

5. SEGURIDAD:
   - Spring Security configurado
   - Usuarios predefinidos:
     * Usuario: admin / Contraseña: admin123
     * Usuario: user / Contraseña: user123

6. VISTAS HTML (Ya existentes):
   - Login.html
   - menu.html
   - Pacientes: lista, nuevo, editar
   - Doctores: lista, nuevo, editar
   - Emergencias: lista, nuevo, editar

CONFIGURACION DE BASE DE DATOS - MYSQL
---------------------------------------
El proyecto está configurado para usar MySQL 8+.

1. CREAR LA BASE DE DATOS:
   Opción A - Usar el script SQL:
   - Ubicación: database/schema.sql
   - Ejecutar: mysql -u root -p < database/schema.sql
   - (Ingrese la contraseña de MySQL cuando se le pida)

   Opción B - Crear manualmente:
   mysql -u root -p
   > CREATE DATABASE hst_db;
   > USE hst_db;
   > (Ejecutar contenido del archivo schema.sql)

2. VARIABLES DE ENTORNO:
   Configurar en el archivo .env o como variables del sistema:
   - DATABASE_URL=jdbc:mysql://localhost:3306/hst_db?useSSL=false&serverTimezone=UTC
   - DATABASE_USERNAME=root
   - DATABASE_PASSWORD=(contraseña de MySQL)

3. SPRING JPA:
   - Configurado con hibernate.ddl-auto=update
   - Las tablas se crean automáticamente al iniciar la aplicación

COMPILAR Y EJECUTAR
--------------------
1. Compilar el proyecto:
   ./mvnw clean install

2. Ejecutar la aplicación:
   ./mvnw spring-boot:run

3. Acceder a la aplicación:
   http://localhost:8080

FLUJO DE LA APLICACIÓN
-----------------------
1. Ingresar a: http://localhost:8080
2. Redirige al login
3. Iniciar sesión con admin/admin123 o user/user123
4. Acceder al menú principal con 3 opciones:
   - Gestión de Pacientes
   - Gestión de Doctores
   - Gestión de Emergencias

ESTRUCTURA DE LA BASE DE DATOS - MYSQL
--------------------------------------
TABLA: pacientes
- id (BIGINT AUTO_INCREMENT PRIMARY KEY)
- nombre, apellido, dni
- fecha_nacimiento, genero, tipo_sangre
- estado_civil, ocupacion
- telefono, direccion
- contacto_emergencia, telefono_emergencia
- caso, alergias, medicamentos
- observaciones, created_at, updated_at

TABLA: doctores
- id (BIGINT AUTO_INCREMENT PRIMARY KEY)
- nombre, apellido, dni
- fecha_nacimiento, genero, estado_civil
- universidad, telefono, email, direccion
- especialidad, numero_licencia
- anos_experiencia, observaciones
- created_at, updated_at

TABLA: emergencias
- id (BIGINT AUTO_INCREMENT PRIMARY KEY)
- nombre_completo
- doctor_asignado
- fecha_hora
- estado_salud
- observaciones, created_at, updated_at
